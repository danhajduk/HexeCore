# Policies Directory

Status: Reference / design guidance

## Purpose

Files in this directory contain policy, specification, and checklist material used for architecture direction and planning.

## Authority Boundary

- These files are **not** the implementation source of truth.
- They may include planned or aspirational behavior that is not yet implemented.

For current implemented behavior, use:
- `docs/core.md`
- `docs/api.md`
- `docs/backend.md`
- `docs/mqtt-contract.md`
- repository code
