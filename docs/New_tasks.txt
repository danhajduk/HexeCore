# Phase 1 Tasks — Naming Abstraction Layer
# Start at Task 620

## Scope

This phase introduces a **central naming abstraction layer** so the platform can present itself as **Hexe AI** externally while still tolerating legacy internal `synthia` usage.

### Goal
Create a stable abstraction for:
- platform identity
- component display names
- backend display text
- frontend branding consumption
- API metadata exposure

### This phase SHOULD
- centralize naming
- reduce hardcoded brand strings
- make UI/backend consume shared identity data
- prepare for later deeper migrations

### This phase should NOT yet
- change MQTT topic roots
- rename API routes
- rename internal package/module paths
- change protocol payload keys
- rename stable runtime identifiers unless display-only
