# Proxied UI Redirect And Link-Generation Contract

Status: Implemented (contract definition)
Last updated: 2026-03-22

## Purpose

Defines how proxied node and addon targets must generate redirects and links so browser navigation remains inside the Core public origin and the mounted proxy prefix.

## Required Behavior

- redirects generated by node/addon backends must preserve the forwarded prefix
- relative redirects should be preferred where possible
- absolute redirects to internal upstream URLs must be blocked or rewritten
- generated links in HTML responses must remain valid under mounted paths

## Rules

- a target must never redirect a browser to its internal `ui_base_url`
- Core may rewrite unsafe `Location` headers when necessary
- targets should support prefix-aware redirect generation
- HTML links and forms should stay inside the mounted public prefix unless intentionally leaving the proxied surface

## Preferred Patterns

- use relative redirects when practical
- use `X-Forwarded-Prefix`, `X-Forwarded-Host`, and `X-Forwarded-Proto` when absolute URL generation is necessary
- generate links from runtime public context rather than upstream private address knowledge

## Not Allowed

- redirects to `http://127.0.0.1:...`, `http://localhost:...`, or LAN/private upstream hosts
- absolute redirects that drop `/nodes/{node_id}/ui/` or `/addons/{addon_id}/`
- HTML navigation elements that point to upstream internal origins instead of the Core public route

## Acceptance Checks

A proxied target satisfies the redirect and link-generation contract only if:

- login redirects remain inside the Core public origin
- app redirects and post-action redirects preserve the mounted prefix
- no redirect leaks internal hosts
- browser navigation remains stable for links rendered from HTML responses

## Failure Examples

Common redirect and link-generation failures:

- a login redirect to `http://localhost:8765/login`
- a post-save redirect to `/dashboard` when the UI is mounted under `/nodes/{node_id}/ui/`
- server-rendered HTML links that point at the upstream base instead of the Core proxy path
